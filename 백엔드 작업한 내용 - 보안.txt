1. 로그인 보안
JWT 토큰 방식
로그인 성공 시 "출입증"을 발급합니다. 이 출입증은 두 가지로 구성됩니다.

Access Token (1시간): 실제 서비스 이용 시 사용하는 출입증
Refresh Token (7일): 출입증이 만료됐을 때 재발급받는 갱신권

출입증은 브라우저의 안전한 공간(HttpOnly Cookie)에 보관되어 외부에서 훔쳐볼 수 없습니다. 갱신권을 한 번 사용하면 즉시 폐기하고 새 갱신권을 발급하기 때문에 누군가 갱신권을 탈취해도 재사용이 불가합니다.
비밀번호를 변경하거나 계정이 정지되면 기존 출입증이 즉시 무효화됩니다.

2단계 인증 (2FA)
로그인 시 비밀번호 외에 이메일로 발송된 6자리 코드를 추가로 입력해야 합니다.

관리자는 반드시 2단계 인증 필수
기업/개인 회원은 선택적으로 활성화 가능
인증 코드는 5분간만 유효


2. 권한 관리
서비스 이용자를 4가지 등급으로 나누어 각 등급에 맞는 기능만 사용할 수 있습니다.
일반 구직자  → 공고 지원, 출퇴근, 이력서 관리만 가능
기업 회원    → 공고 등록, 지원자 관리, 정산 가능
매니저      → 기업 소속으로 일부 기업 기능 사용 가능
관리자      → 전체 회원/공고/정산 관리 가능
예를 들어 구직자가 기업 기능(공고 등록)을 시도하면 즉시 차단됩니다. 또한 본인의 공고, 계약서, 지원서만 접근할 수 있어 타인의 데이터를 볼 수 없습니다.

3. 과부하/어뷰징 방지 (Rate Limiting)
같은 사용자나 IP에서 짧은 시간에 너무 많은 요청을 보내면 자동으로 차단합니다. 예를 들어 경쟁사가 서버에 무차별 요청을 보내 서비스를 마비시키려 해도 방어할 수 있습니다. 구독 플랜에 따라 허용 요청 횟수가 다르게 적용됩니다.

4. 해킹 공격 방어
XSS 공격 방어
XSS란 악성 스크립트를 웹사이트에 심어 사용자 정보를 탈취하는 공격입니다.
예를 들어 누군가 공고 제목에 아래와 같은 코드를 입력한다면:
<script>document.cookie를 해커 서버로 전송</script>
이 코드가 다른 사용자 화면에 표시되는 순간 해당 사용자의 로그인 정보가 탈취됩니다.
Promoter는 이를 3중으로 방어합니다:

입력값에서 HTML 코드 자동 제거 (jsoup)
로그인 토큰을 스크립트로 접근 불가한 공간에 보관 (HttpOnly Cookie)
CSP 헤더로 허용된 스크립트만 실행 가능하도록 제한

클릭재킹 방어
Promoter 화면을 다른 사이트 안에 몰래 숨겨놓고 사용자가 모르게 클릭하도록 유도하는 공격입니다. X-Frame-Options: DENY 설정으로 다른 사이트에 Promoter 화면 삽입을 원천 차단합니다.
SQL Injection 방어
데이터베이스에 악성 명령어를 심어 정보를 탈취하는 공격입니다. Promoter는 모든 데이터베이스 쿼리를 안전한 방식(파라미터 바인딩)으로 처리하여 이 공격을 원천 차단합니다.

5. 개인정보 보호
비밀번호 암호화
비밀번호는 BCrypt 방식으로 암호화하여 저장합니다. 데이터베이스가 유출되더라도 비밀번호를 원래 값으로 되돌릴 수 없습니다.
개인정보 최소 노출
API 응답 시 꼭 필요한 정보만 제공하고 민감한 정보(비밀번호, 개인식별번호 등)는 절대 노출하지 않습니다.
회원 탈퇴 처리
회원 탈퇴 요청 후 30일간 데이터를 보관하다가 자동으로 익명화 처리합니다. 탈퇴 후에도 근무 기록 등 법적 의무 데이터는 안전하게 보존됩니다.

6. 파일 업로드 보안
파일 업로드 시 허용된 파일 형식(이미지, PDF 등)만 받고 크기를 제한(최대 10MB)하여 악성 파일 업로드를 차단합니다.

7. GPS 위치 검증
퇴근 체크아웃 시 근무지 반경 200m 이내에서만 가능하도록 GPS 위치를 검증합니다. 원격에서 허위 퇴근 처리를 방지합니다.
